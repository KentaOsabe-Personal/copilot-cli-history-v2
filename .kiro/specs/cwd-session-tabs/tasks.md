# 実装計画

- [x] 1. 作業ディレクトリ分類の基盤を作る
- [x] 1.1 作業ディレクトリ別タブの分類・表示名・絞り込み規則を実装する
  - 成功状態のセッション集合から `すべて`、作業ディレクトリ別、`ディレクトリ未設定` のタブを導出する
  - `cwd` の前後空白を除いて同じ作業ディレクトリとして扱い、未設定値や空白値は未設定タブへまとめる
  - タブの主ラベルは最後のディレクトリ名にし、同一 basename は最短一意の親パス文脈で区別できるようにする
  - 選択タブに対応するセッションだけを既存の相対順で返し、存在しない選択タブは `すべて` に補正される
  - 完了時には、取得済みセッション集合だけから件数付きタブと表示対象セッションを再現できる
  - _Requirements: 1.1, 1.2, 1.3, 1.4, 1.5, 2.1, 2.2, 2.3, 2.4, 2.5, 3.2, 3.3, 3.4, 5.1, 5.2, 5.3, 6.2, 6.3, 6.4_

- [x] 1.2 作業ディレクトリ分類規則の単体テストを追加する
  - 各 test case の直前に `概要・目的`、`テストケース`、`期待値` のコメントを置く
  - 全件タブ、trim 後の grouping、未設定 grouping、basename 表示、同一 basename の文脈付与を検証する
  - タブ選択時の相対順維持と、存在しない選択タブの `すべて` 補正を検証する
  - 完了時には、分類規則の主要な境界条件が Vitest で失敗なく確認できる
  - _Requirements: 1.2, 1.3, 1.4, 1.5, 2.1, 2.2, 2.3, 2.4, 2.5, 3.4, 5.2, 5.3, 6.4_

- [x] 2. 操作可能なタブ UI と検索案内を整える
- [x] 2.1 (P) 作業ディレクトリ別タブをアクセシブルなタブ UI として提供する
  - タブ一覧と各タブに範囲・件数・完全パスが分かるアクセシビリティ名を付ける
  - クリック、タップ相当、左右キーで選択タブを切り替え、選択状態を視覚と支援技術へ伝える
  - 長いパスや多いタブでもページ全体の横スクロールやテキスト重なりが起きない表示にする
  - 完了時には、件数付きのタブ列がローカル横スクロール内で操作でき、選択タブが対応するパネルを参照する
  - _Requirements: 1.1, 1.5, 2.3, 2.4, 2.5, 4.1, 4.2, 4.3, 4.4, 4.5_
  - _Boundary: SessionDirectoryTabs_
  - _Depends: 1.1_

- [x] 2.2 (P) 検索フォームの補助説明を作業ディレクトリ別タブの文脈へ更新する
  - 実行ディレクトリを検索対象として案内する文言を外す
  - 検索結果の中で作業ディレクトリを一覧タブで切り替えられることが分かる説明にする
  - 完了時には、検索条件の意味を変えずに、作業ディレクトリ切替の案内だけが表示文言へ反映される
  - _Requirements: 5.4, 6.5_
  - _Boundary: SessionSearchForm_

- [x] 2.3 (P) 作業ディレクトリ別タブ UI のコンポーネントテストを追加する
  - 各 test case の直前に `概要・目的`、`テストケース`、`期待値` のコメントを置く
  - クリック、左右キー、循環移動、選択状態、タブとパネルの関連付けを検証する
  - basename、必要な親パス文脈、完全パス、件数、未設定タブの accessible name を検証する
  - 完了時には、タブ UI の操作とアクセシビリティ契約が Testing Library で確認できる
  - _Requirements: 1.5, 2.3, 2.4, 2.5, 4.1, 4.2, 4.3, 4.4, 4.5_
  - _Boundary: SessionDirectoryTabs tests_
  - _Depends: 2.1_

- [x] 2.4 (P) 検索フォーム補助説明のコンポーネントテストを更新する
  - 各 test case の直前に `概要・目的`、`テストケース`、`期待値` のコメントを置く
  - 補助説明が `cwd` 検索案内ではなく、一覧タブによる作業ディレクトリ切替案内になっていることを検証する
  - 完了時には、検索フォームの表示文言が検索 API 契約を変えないことをテストで確認できる
  - _Requirements: 5.4, 6.5_
  - _Boundary: SessionSearchForm tests_
  - _Depends: 2.2_

- [x] 3. セッション一覧成功表示へタブを統合する
- [x] 3.1 成功状態のセッション集合からタブを再構築し、選択状態を補正する
  - 成功状態でだけ作業ディレクトリ別タブを導出し、loading、empty、error では表示しない
  - 日付範囲、検索語、同期後の再読み込みで新しい成功結果が表示されたら、その集合からタブを作り直す
  - 選択中の作業ディレクトリが新しい集合に存在しない場合は `すべて` に戻す
  - 完了時には、タブ選択が page-local state に閉じ、API query、URL、永続化状態へ混ざらない
  - _Requirements: 1.1, 1.4, 3.1, 3.2, 3.3, 3.4, 3.5, 5.1, 5.5, 6.1, 6.2, 6.3, 6.4, 6.5_
  - _Depends: 1.1, 2.1_

- [x] 3.2 選択中タブに応じた一覧と補助行を描画する
  - `すべて` は現在条件の全セッション、作業ディレクトリ別タブは該当セッション、未設定タブは未設定セッションだけを既存順で表示する
  - 選択中の作業ディレクトリでは完全パスを一覧上部で確認でき、`すべて` と未設定は完全パスと誤認しない補助表示にする
  - 一覧領域を選択タブと関連付くパネルとして描画し、支援技術が選択範囲を追えるようにする
  - 完了時には、タブを切り替えるたびに同じ一覧領域内のカードだけが選択範囲に合わせて変わる
  - _Requirements: 1.2, 1.3, 1.5, 2.3, 2.5, 3.1, 4.3, 4.4, 5.2, 5.3, 6.1_
  - _Depends: 3.1_

- [x] 3.3 検索結果と既存状態表示の契約を維持する
  - 検索語が適用された成功結果では、その検索済み集合だけからタブと一覧を切り替える
  - 検索成功時の説明文を、作業ディレクトリは一覧タブで切り替える文脈へ合わせる
  - 検索 empty、通常 empty、loading、error の既存表示を維持し、空のタブ列を表示しない
  - 完了時には、検索・日付・同期の既存操作後もタブ追加前と同じ状態表示契約が保たれる
  - _Requirements: 3.2, 3.3, 3.5, 5.1, 5.2, 5.3, 5.4, 5.5, 6.1, 6.5_
  - _Depends: 3.1, 3.2_

- [x] 4. ページ統合の回帰テストを追加する
- [x] 4.1 成功状態のタブ表示と一覧切替をページテストで検証する
  - 各 test case の直前に `概要・目的`、`テストケース`、`期待値` のコメントを置く
  - 成功状態で `すべて` と作業ディレクトリ別タブが表示され、件数と一覧カードが選択範囲に一致することを検証する
  - タブと一覧パネルが id と aria 属性で関連付くことを検証する
  - 完了時には、成功表示での主要なタブ切替体験がページ単位で確認できる
  - _Requirements: 1.1, 1.2, 1.3, 1.5, 4.3, 4.4_
  - _Depends: 3.2_

- [x] 4.2 タブ切替が検索・日付・同期の既存操作を発火しないことを検証する
  - 各 test case の直前に `概要・目的`、`テストケース`、`期待値` のコメントを置く
  - タブ切替時に日付範囲、検索語、同期後 refresh の操作が呼ばれず、表示条件も変わらないことを検証する
  - 選択中ディレクトリの完全パス補助行が、タブ選択に追従して更新されることを検証する
  - 完了時には、タブ切替が取得済みセッション集合の local filtering に閉じることをテストで確認できる
  - _Requirements: 2.3, 3.1, 6.1, 6.2, 6.3, 6.5_
  - _Depends: 3.2_

- [x] 4.3 検索結果、再取得補正、非成功状態の回帰をページテストで検証する
  - 各 test case の直前に `概要・目的`、`テストケース`、`期待値` のコメントを置く
  - 検索語ありの成功結果で、検索済み集合だけから `すべて` と作業ディレクトリ別タブが切り替わることを検証する
  - 新しい成功結果で選択中タブが消えた場合に `すべて` へ補正されることを検証する
  - loading、empty、error、検索 empty ではタブが表示されず、既存の状態表示が残ることを検証する
  - 完了時には、再取得と検索の境界条件でもタブ状態が利用者に残留表示されない
  - _Requirements: 3.2, 3.3, 3.4, 3.5, 5.1, 5.2, 5.3, 5.5, 6.1_
  - _Depends: 3.1, 3.3_

- [x] 5. 対象テストと既存フロントエンド検証を実行し、回帰を解消する
  - 作業ディレクトリ分類、タブ UI、検索フォーム、ページ統合のテストを実行する
  - 既存の frontend lint / test / build の標準コマンドで、型・lint・ビルド回帰がないことを確認する
  - backend endpoint、API response shape、server-side aggregation、pagination、永続化を変更していないことを差分で確認する
  - 完了時には、追加したテストと既存品質ゲートが通り、作業ディレクトリ別タブの実装だけが差分として残る
  - _Requirements: 6.1, 6.2, 6.3, 6.4, 6.5_
